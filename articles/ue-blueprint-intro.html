Unreal Engine 蓝图入门指南

从零开始了解 UE 的可视化脚本系统，理解节点、事件和执行逻辑的基本概念。

什么是蓝图？

蓝图（Blueprint）是 Unreal Engine 的可视化脚本系统。它允许开发者通过连接节点来实现游戏逻辑， 而不需要编写一行 C++ 代码。对于独立游戏开发者来说，蓝图是快速原型和迭代的核心工具。

蓝图不是「简化版编程」，它是一套完整的逻辑系统，可以完成从简单交互到复杂 AI 的几乎所有游戏功能。 很多商业游戏的核心玩法就是纯蓝图实现的。

💡 为什么要学蓝图？

即使你计划用 C++ 开发，蓝图也是 UE 工作流中不可或缺的部分。很多功能（动画状态机、UI、关卡流程） 最适合用蓝图实现。学会蓝图能让你更高效地使用 UE。

蓝图的基本概念

1. 节点（Node）

每个节点代表一个操作或功能。比如「打印字符串」「移动角色」「播放动画」等。 节点之间通过「引脚」（Pin）连接，数据从一个节点流向下一个节点。

2. 事件（Event）

事件是蓝图的起点。最常见的是 Event Begin Play（游戏开始时执行） 和 Event Tick（每一帧执行）。你也可以自定义事件来响应玩家输入或游戏状态变化。

3. 执行引脚与数据引脚

蓝图中有两种引脚：

  • 执行引脚（白色箭头）：控制代码的执行顺序，像一条流水线
  • 数据引脚（彩色圆点）：传递具体的数据值，比如数字、文本、对象引用

4. 变量（Variable）

变量用来存储数据，和编程中的变量概念一样。UE 中的变量有类型（整数、浮点数、布尔值、字符串、对象引用等）， 可以在蓝图面板中创建和管理。 

// 等价的 C++ 代码
float PlayerHealth = 100.0f;
bool bIsAlive = true;
FString PlayerName = TEXT("歪歪");

创建你的第一个蓝图

步骤一：创建蓝图类

在内容浏览器中右键 → Blueprint Class → 选择 Actor 作为父类。 命名为 BP_MyFirstActor。

步骤二：添加组件

打开蓝图编辑器，在 Components 面板点击 Add → 选择 Static Mesh。 在 Details 面板中选择一个网格体（比如 Cube）。

步骤三：编写逻辑

在事件图表（Event Graph）中：

  1. 找到 Event Begin Play 节点
  2. 右键搜索 Print String，添加节点
  3. 连接 Event Begin Play 的执行引脚到 Print String
  4. 在 Print String 的输入框填入 Hello Unreal!
🎮 运行测��

把蓝图拖入关卡，按 Play 运行。屏幕左上角会显示「Hello Unreal!」。 恭喜，你完成了第一个蓝图程序！

蓝图学习的推荐路线

  1. 基础操作 — 节点创建、连接、变量使用、条件分支
  2. Actor 交互 — 生成与销毁 Actor、碰撞检测、射线检测
  3. 动画蓝图 — 状态机、动画混合、通知（Notify）
  4. UI 系统 — UMG 控件、绑定、动画
  5. AI 系统 — 行为树、黑板、EQS
  6. 优化与架构 — 蓝图通信（接口、事件分发器）、性能优化

常见问题

蓝图和 C++ 怎么选？

不是二选一的关系。推荐做法是：用 C++ 写底层系统和性能敏感的代码， 用蓝图做玩法逻辑和快速迭代。UE 的设计就是让两者无缝协作的。

蓝图性能会不会很差？

对于大多数游戏来说，蓝图性能完全够用。真正需要优化的地方可以随时转为 C++。 不要过早优化，先让游戏跑起来更重要。
